在每次构建前执行的 Groovy 脚本。脚本可以访问以下绑定变量：

  • job - 当前 Jenkins 任务
  • jenkins - Jenkins 实例
  • env - 环境变量映射
  • params - 构建参数映射
  • build / currentBuild - 当前构建（如果可用）

构建参数也可以作为直接变量使用。例如，如果有一个名为 DEPLOY_ENV 的参数， 可以通过 DEPLOY_ENV 或 params.get('DEPLOY_ENV') 访问。

脚本必须返回一个包含以下键的 Map：

  • blockBuild（布尔值，默认：false）- 如果为 true，阻止构建并仅显示弹窗
  • showPopup（布尔值，默认：false）- 如果为 true，显示弹窗通知
  • popupContent（字符串）- 在弹窗中显示的消息
  • popupTitle（字符串，默认："Build Notification"）- 弹窗标题

示例： 

if (job.isBuilding()) {
    return [blockBuild: true, showPopup: true,
            popupContent: "任务 ${job.name} 正在构建中！",
            popupTitle: "构建警告"]
} else {
    return [blockBuild: false, showPopup: false, popupContent: '']
}

注意：请使用 Map 字面量语法 [key: value]，而不是方法调用如 popupContent('xxx')。

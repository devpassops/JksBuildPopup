When enabled, the Groovy script will be executed in a Jenkins Script Security sandbox. This restricts access to potentially dangerous operations for security.

What the sandbox restricts:

  • File system access (reading/writing files)
  • Network operations (HTTP requests, sockets)
  • Process execution (running external commands)
  • Direct access to Jenkins internal APIs

What is allowed in sandbox:

  • Access to job, build, and parameter information
  • String manipulation and basic Groovy operations
  • Approved method calls (whitelisted by administrators)

Recommendation: Enable this option unless you trust the script authors and need access to restricted operations. Scripts that require restricted operations will fail with a clear error message indicating which method was rejected.

Administrators can approve specific method calls through Jenkins → Script Console or by configuring script approval in the system configuration.
